# Экспериментальная оценка

## Цель

Этот документ описывает минимальный воспроизводимый набор метрик для оценки прототипа СППР TravelContext перед защитой.

## Сценарий запуска

Команда:

```bash
cd /Users/teamofey/magistr-diploma/backend
UV_CACHE_DIR=../.uv-cache uv run python -m app.tools.evaluate_recommender --json
```

Скрипт:

- строит набор синтетических сценариев по нескольким городам;
- формирует расширенный профиль пользователя;
- генерирует рекомендации;
- собирает маршрут с учетом стартовой точки отеля, если она есть в каталоге;
- считает агрегированные метрики.

## Метрики

### `precision@k`

Доля объектов в топ-k, которые действительно совпали с выбранными интересами пользователя (`matched_interests` не пустой).

### `coverage`

Доля категорий, покрытых в выдаче, относительно всего набора категорий, доступных в каталоге города. Метрика показывает, насколько система не схлопывается в один тип объектов.

### `budget-fit`

Бинарная метрика: `1`, если итоговый маршрут укладывается в бюджет пользователя, и `0`, если нет.

### `time-fit`

Бинарная метрика: `1`, если маршрут укладывается в общий временной лимит поездки, и `0`, если нет.

### `explanation usefulness`

Эвристика по тексту объяснения:

- `1.0`, если объяснение содержит и ссылку на интерес пользователя, и ссылку на ограничения;
- `0.5`, если есть только одно из двух;
- `0.0`, если объяснение слишком общее.

### `context robustness`

Стабильность топа рекомендаций после мягкого изменения контекста. В текущем скрипте увеличиваются бюджет и допустимая дистанция, после чего считается Jaccard overlap между двумя топами.

## Как интерпретировать результаты

- `precision@k >= 0.6` означает, что топ выдачи в основном попадает в интересы пользователя.
- `coverage >= 0.3` показывает, что маршрут не состоит только из одной категории.
- `budget-fit` и `time-fit` должны быть близки к `1.0` для спокойных и умеренных сценариев.
- `explanation usefulness >= 0.5` означает, что объяснения уже пригодны для демонстрации explainable DSS.
- `context robustness` не должен быть слишком низким: сильная нестабильность говорит о переобучении на мелких сдвигах профиля.

## Ограничения текущей методики

- Метрики считаются на seed-каталоге и не заменяют живое пользовательское исследование.
- `explanation usefulness` пока эвристическая и не основана на экспертной разметке.
- `context robustness` измеряет устойчивость только к мягким числовым изменениям, а не к полной смене целей поездки.
